# CLAUDE.md

> 本文件为 **Claude Code** 在本仓库工作时的专用指引。内容基于 `AGENTS.md`
> 精简改写，聚焦 Claude Code 的工具链与工作流约定。若本文件与 `AGENTS.md`
> 冲突，以本文件为准；未覆盖的条目回退到 `AGENTS.md`。

## 一、仓库性质

本仓库**不是软件工程项目**，而是一份按主题整理的**中医古籍参考资料库**
（全部为 `.epub`）。目的是为 Claude Code（及人类读者）在回答中医问题时
提供可溯源的原始依据。

由于没有源代码，Claude Code 常见的「构建 / 测试 / Lint / CI」流程在此
**不适用**。本文件聚焦：资料分类、查阅策略、引用规范、工具使用约束。

## 二、知识分层（道—法—方—药—术）

> 《内经》讲「为什么」，《本草经》讲「用什么」，《伤寒》《金匮》讲「怎么辨、
> 怎么开方」，《针灸大成》讲「怎么扎、怎么灸」。

| 层级   | 回答的问题         | 目录         | 代表典籍                 |
| ---- | ------------- | ---------- | -------------------- |
| 道·规律 | 人体运行的根本规律     | `病理/`      | 《黄帝内经》               |
| 药·药性 | 每味药的性味归经与功效   | `药理/`      | 《神农本草经》《难经》          |
| 法·辨证 | 外感 / 内伤如何辨证施治 | `处方思路/`    | 《伤寒杂病论》《金匮要略》        |

## 三、强制前置规则（MUST）

在回答任何中医相关问题前，Claude Code 必须按顺序完成以下动作，任何一步都
不得跳过：

1. **必须先查典籍**：根据问题层级定位目录（见下节工作流），实际打开对应
   `.epub` 检索相关条目。未经检索不得作答。
2. **epub 读取失败时必须明示**：若因缺少 EPUB 解析器等原因无法读取原文，
   必须在回答中**显式声明「未查证原文」**；可通过网络搜索补充典籍未提及
   的内容，但**需要向用户说明**。
3. **禁止伪托出处**：未实际查阅到的条文、篇目、原文引用一律不得出现在
   回答中。笼统写「参见《XX》」而未核对原文，视同伪托，属于「臆造」。
4. **食疗 / 养生 / 日常保健类问题同样适用**：不因问题「偏生活化」而降低
   查证标准。凡涉及性味、归经、功效、配伍的论断，均须回到 `药理/` 等
   目录核对。

## 四、Claude Code 工作流（回答中医问题时）

1. **判定问题层级**，选择目录：
   - 原理 / 为什么 → `病理/`（《黄帝内经》）
   - 药性 / 配伍 → `药理/`（《神农本草经》；脏腑脉法疑难兼查《难经》）
   - 辨证 / 开方 → `处方思路/`（外感《伤寒论》、内伤《金匮要略》）
   - 经络 / 针灸 → `穴位经络/`（《针灸大成》+ 倪海厦文稿）
   - 古文字词 → `古文翻译/`（《古汉语常用字字典》）
2. **遵循规律**：依据该目录核心规律推演答案，不脱离经典体系凭空发挥。
3. **标注出处**：涉及原理、条文、方剂、穴位主治时引用书名；能指到篇章
   （如《素问·阴阳应象大论》《伤寒论·太阳病篇》）更佳。
4. **跨目录协同**：复杂问题常需多目录协同。例如外感处方：
   - `病理/` 查表里寒热判断依据
   - `处方思路/` 查六经归类与对应方
   - `药理/` 核对每味药性味归经
   - 必要时 `古文翻译/` 核对字词
5. **拒绝臆造**：所查目录未涵盖该问题时，明确告知来源不足，而非编造；
   **未经核对而笼统引用书名同样属于臆造**。

## 五、Claude Code 工具使用规则（本仓库专用）

### 5.1 文件检索（glob / grep / list_dir）

- 文件名含中文与括号，shell 传参**必须加引号**（单引号或双引号）。
- 搜索范围默认限定于五大分类目录，排除 `.git/`、`node_modules/` 等。
- `list_dir` 列目录优先，不要无约束 `find` 全盘扫描。

### 5.2 读取 EPUB（read_file 不适用）

- `read_file` 工具仅能读取纯文本，**不能直接读取 `.epub`**。
- 需要读原文时，通过 `bash` 调用 Python + `ebooklib`（或 `unzip` 取出
  XHTML）解析；可尝试使用 `.venv` 下的工具或其他方式读取 EPUB 内容。
  示例（只读，不得改动原文件）：

  ```bash
  python3 - << 'PY'
  from ebooklib import epub
  book = epub.read_epub("病理/黄帝内经（全二册）.epub")
  for item in book.get_items():
      if item.get_type() == 9:  # DOCUMENT
          print(item.get_name())
  PY
  ```

- 解析产生的临时文本须写入 `./tmp/` 子目录，不要散落在仓库根。

### 5.3 写文件 / 编辑文件

- **原始 EPUB 只读**：禁止 `write_file` / `edit_file` / `mv` / `rm`
  任何 `.epub` 文件，亦不得重新打包或对外再分发。
- 新建的衍生物（笔记、目录索引、解析脚本等）统一放 `notes/`、`scripts/`、
  `tmp/` 等新目录，避免与原始资料混放。

### 5.4 破坏性操作

- 触发 `rm -rf`、强制覆盖原资料、批量重命名前**必须先与用户确认**。
- 不对 `.git/` 做任何强制操作（`push --force`、`reset --hard` 等）。

### 5.5 MCP / 远程工具

- 本仓库所有文件 IO 使用**本地工具**（bash / read_file / write_file /
  edit_file / glob / grep / list_dir）。
- 需要查证现代文献或校对译文时，可通过网络搜索，但引用仍须回到
  本仓库原始典籍为准，联网内容仅作辅助。

## 六、推荐阅读顺序（零基础）

**原理 → 药物 → 临床 → 操作**

1. `病理/`《黄帝内经·素问》选读 — 建立阴阳五行、藏象经络框架
2. `药理/`《神农本草经》选读 — 理解药性语言
3. `处方思路/`《伤寒论》 — 学会用六经归类外感
4. `处方思路/`《金匮要略》 — 扩展到内伤杂病脏腑辨证
5. `穴位经络/`《针灸大成》 — 掌握经络腧穴与针刺操作
6. `古文翻译/`《古汉语常用字字典》 — 随查随用

## 七、引用格式约定

回答中标注出处时，建议形式：

- **书名**：`《黄帝内经》`、`《伤寒论》`、`《针灸大成》`
- **篇章**：`《素问·阴阳应象大论》`、`《伤寒论·太阳病上篇》`
- **本仓库文件路径**（必要时）：`病理/黄帝内经（全二册）.epub`

避免在同一句里混用「书名」与「文件路径」造成冗余；一般行文用书名+篇章，
需明确资料来源时再附路径。
